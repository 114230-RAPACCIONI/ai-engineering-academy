# ADR-003 — Documentación como artefacto de producto

## Status

Accepted

## Contexto

ZUZU enseña ingeniería moderna. La documentación no puede ser secundaria — para el producto **y** para la práctica del learner.

## Decisión

Los artefactos son first-class: requirements, decisions y reviews en Practice Projects son evidencia de capability — no un wiki de Project OS.

## Consecuencias

+ Trazabilidad y hábitos de aprendizaje  
− Mantener el alcance de artefactos liviano en el MVP
